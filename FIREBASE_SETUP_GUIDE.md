# Инструкция по настройке и проверке Firebase для Pixel Word Hunter

## 1. Архитектура хранения данных

### Структура Firestore Database

Данные хранятся в коллекции `users`, где каждый документ идентифицируется по `UID` пользователя:

```
firestore
└── users/
    └── {userUid}/
        ├── createdAt: timestamp
        ├── updatedAt: timestamp
        ├── lastSync: timestamp
        ├── xp: number (общее количество очков опыта)
        └── progress: map {
              {wordEng}: {
                mastery: number,
                lastSeen: timestamp,
                correctCount: number,
                incorrectCount: number
              },
              ...
            }
```

### Пример документа пользователя:

```json
{
  "createdAt": "2025-01-15T10:30:00.000Z",
  "updatedAt": "2025-01-15T14:45:00.000Z",
  "lastSync": "timestamp(2025-01-15T14:45:00.000Z)",
  "xp": 1250,
  "progress": {
    "apple": {
      "mastery": 3,
      "lastSeen": 1736949600000,
      "correctCount": 5,
      "incorrectCount": 1
    },
    "banana": {
      "mastery": 2,
      "lastSeen": 1736949500000,
      "correctCount": 3,
      "incorrectCount": 2
    }
  }
}
```

## 2. Как проверить правильность настройки

### Шаг 1: Проверка в Firebase Console

1. Откройте [Firebase Console](https://console.firebase.google.com/)
2. Выберите проект `pixelwordhunter`
3. Перейдите в раздел **Firestore Database**
4. Проверьте коллекцию `users`:
   - Каждый пользователь должен иметь документ с UID как именем документа
   - UID можно найти в разделе **Authentication** → **Users**

### Шаг 2: Проверка для конкретного пользователя (например, becoda5573@pckage.com)

1. В Firebase Console перейдите в **Authentication** → **Users**
2. Найдите пользователя с email `becoda5573@pckage.com`
3. Скопируйте его **User UID** (выглядит как `abc123XYZ...`)
4. Перейдите в **Firestore Database**
5. Найдите документ `users/{скопированный_UID}`
6. Проверьте наличие полей:
   - `progress` - карта с прогрессом по словам
   - `xp` - числовое значение опыта
   - `lastSync` - время последней синхронизации

### Шаг 3: Проверка изоляции данных между пользователями

1. Войдите под первым пользователем (например, `user1@example.com`)
2. Пройдите несколько вопросов в игре
3. Выйдите из аккаунта (кнопка Logout)
4. Войдите под вторым пользователем (например, `user2@example.com`)
5. Убедитесь, что:
   - Прогресс второго пользователя не содержит данных первого
   - XP начинается с нуля (или с предыдущего значения второго пользователя)
   - В консоли браузера видно загрузку данных только для текущего UID

## 3. Логирование для отладки

Приложение выводит подробные логи в консоль браузера:

```javascript
[Auth] Auth state changed: abc123XYZ...
[Auth] User authenticated, loading progress from Firestore...
[Storage] Cloud sync successful
[Storage] XP loaded from server: 150
[App] Progress applied: 25 words, XP: 150
[XP] Added 10 XP atomically for user abc123XYZ...
[XP Sync] Real-time listener established for user: abc123XYZ...
```

### Как включить подробное логирование:

1. Откройте DevTools в браузере (F12)
2. Перейдите на вкладку **Console**
3. Фильтруйте логи по префиксам:
   - `[Auth]` - события аутентификации
   - `[Storage]` - операции загрузки/сохранения
   - `[XP]` - операции с очками опыта
   - `[XP Sync]` - синхронизация XP между вкладками

## 4. Проверка работы синхронизации

### Тест 1: Загрузка данных при входе

1. Откройте DevTools Console
2. Войдите в аккаунт с email/паролем
3. Ожидаемые логи:
   ```
   [Auth] Auth state changed: {UID}
   [Auth] User authenticated, loading progress from Firestore...
   [Storage] Cloud sync successful
   [Storage] XP loaded from server: {значение}
   [App] Progress applied: {число} words, XP: {значение}
   ```

### Тест 2: Сохранение данных во время игры

1. Пройдите 1-2 вопроса в игре
2. Подождите 2 секунды (debounce перед сохранением)
3. Ожидаемые логи:
   ```
   [XP] Added 10 XP atomically for user {UID}
   [Storage] Cloud saved
   ```

### Тест 3: Синхронизация между вкладками

1. Откройте приложение в двух вкладках браузера
2. Войдите под одним аккаунтом в обеих вкладках
3. В первой вкладке пройдите вопрос (+10 XP)
4. Во второй вкладке должно автоматически обновиться XP без перезагрузки
5. Ожидаемые логи во второй вкладке:
   ```
   [XP Sync] XP updated from server: {новое_значение}
   ```

## 5. Правила безопасности Firestore

Рекомендуемые правила безопасности для коллекции `users`:

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // Пользователи могут читать/записывать только свои данные
    match /users/{userId} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }
  }
}
```

### Как установить правила:

1. Firebase Console → **Firestore Database** → **Rules**
2. Вставьте приведённые выше правила
3. Нажмите **Publish**

## 6. Возможные проблемы и решения

### Проблема 1: Данные не загружаются при входе

**Симптомы:**
- В логах видно `[Storage] No user document found`
- Прогресс остаётся пустым после входа

**Решение:**
- Документ создастся автоматически при первом сохранении прогресса
- Пройдите 1-2 вопроса в игре для触发 сохранения

### Проблема 2: Данные одного пользователя видны другому

**Симптомы:**
- При входе под разным аккаунтом отображается чужой прогресс

**Возможные причины:**
- Неправильная привязка к UID в коде
- Кэширование в localStorage

**Решение:**
- Проверьте, что в логах отображается правильный UID
- Очистите localStorage: `localStorage.clear()`
- Убедитесь, что правила безопасности Firestore настроены правильно

### Проблема 3: XP не синхронизируется между вкладками

**Симптомы:**
- В одной вкладке XP обновляется, в другой — нет

**Решение:**
- Проверьте, что real-time listener активен: `[XP Sync] Real-time listener established`
- Убедитесь, что используется один и тот же аккаунт в обеих вкладках
- Проверьте права доступа в Firestore Rules

## 7. Экспорт и импорт данных

### Экспорт прогресса:

1. Настройки → **Export Data**
2. Скачается JSON-файл с резервной копией

### Импорт прогресса:

1. Настройки → **Import Data**
2. Выберите ранее экспортированный JSON-файл
3. Приложение перезагрузится с новыми данными

## 8. Сброс прогресса

Для полного сброса прогресса пользователя:

1. Firebase Console → **Firestore Database**
2. Найдите документ пользователя: `users/{UID}`
3. Удалите документ или очистите поле `progress`

Или через интерфейс приложения:
- Настройки → **Reset Progress** → Подтвердить

## 9. Мониторинг использования

### Firestore Usage:

1. Firebase Console → **Firestore Database** → **Usage**
2. Отслеживайте:
   - Количество чтений/записей
   - Размер хранилища
   - Количество подключений

### Authentication Logs:

1. Firebase Console → **Authentication** → **Users**
2. Фильтруйте по дате регистрации
3. Отслеживайте активных пользователей
